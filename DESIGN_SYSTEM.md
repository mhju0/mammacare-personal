**cream/sage(오트 크림 배경 + 세이지 그린 브랜드)가 앱 전체의 셸이다.** — 2026-07-15 오너 확정.
2026-07-13의 warm-kr(페일 로즈 + 코랄 + 딥 테라코타) 방향에서 피벗했다(오너가 concept 렌더를 보고 결정). `--warm-*` 토큰은 **이름을 유지한 채 값만** cream/sage로 리튠(`theme.css`) → 기존 마이그레이션 화면이 그대로 새 팔레트를 상속.
- IA: 하단 **4탭 홈/재료/리포트/설정**(`Layout.tsx`). 관찰은 진행중 테스트 카드에서 진입(`/observe` 라우트 유지), 프로필은 헤더 아이콘 + 설정에서 진입.
- 스탬프 언어(잉크 스탬프 재료 뱃지)는 도감/리포트 코드에 남아 있으나 세이지 톤으로 리컬러됨. 유지/제거는 후속(CP4) 결정.
- night-watch(다크 모드)는 **보류** — cream 시스템 light-only 유지(`.dark` 블록은 레거시 값 그대로, 미사용).
- 이전 warm-kr 로즈/테라코타 방향과 근거 시안(`docs/mocks/hybrid/`, `warm-kr/`)은 히스토리로 남긴다.
- **새 작업은 `--warm-*` + 시맨틱 상태 토큰만 쓴다. 일회성 하드코딩 색 금지.** (기존 하드코딩은 CP3에서 값-스왑으로 팔레트에 맞춰뒀고, 토큰화는 후속 과제.)

변하지 않는 핵심 원리: **알레르기 상태 = 신호등.** 색이 곧 기능이다.
- 안전/테스트 완료 → 초록(세이지) · 테스트 중/대기 → 앰버(허니) · 반응 확정 → 테라코타
- 신호등 시맨틱 토큰(safe/testing/reaction)은 그대로 쓴다. 색 단독 의미 전달 금지(아이콘+텍스트 병행)도 불변.
- **반응 색 (2026-07-15, 오너 승인 · adversarial review PASS):** 신호등을 3단계로 뚜렷이 분리 — 테스트중 골드(`--testing-fg #B0761F`) < 주의 번트오렌지(`--caution-fg #C05621`) < 반응 레드테라코타(`--reaction-fg #A8432B`, `--destructive`도 상속). 경보 레드(`#C0392B`)보다 차분하지만 스케일에서 가장 진하고 붉어 가장 긴급하게 읽힘. 의미는 아이콘+텍스트 병행으로 보존(색 단독 금지 불변).

- 그림자: `--shadow-warm`(소프트 섀도, `shadow-warm` 유틸). 카드가 보더 대신 이 그림자에 기댄다.
- warm(cream/sage) 시스템은 **light-only**(`.dark` 변형 미사용).
- 브랜드(딥 세이지) vs CTA(세이지) 역할 분리를 지킬 것.
- **후속(알려진 것):** 로고/워드마크 이미지 에셋(`mamma_9.webp` 등)은 구 컬러풀 팔레트 → cream/sage용 재디자인 필요(코드 밖 에셋).
