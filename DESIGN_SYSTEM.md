# DESIGN_SYSTEM — MammaCare (personal)

## 방향 (2026-07 개정 · 2026-07-13 하이브리드 확정)
**warm-kr(페일 로즈 + 코랄 + 딥 테라코타)이 앱 전체의 셸이다.** 여기에 **스탬프 언어(음식 여권 컨셉)를 도감·리포트 두 화면에만** 얹는 하이브리드가 오너 확정 방향.
- 셸 기준 시안: `docs/mocks/hybrid/home.png` (기존 `warm-kr/` 시안의 진화판 — 신호등 3색 점 교정 + 한글 5탭 홈/도감/관찰/리포트/프로필)
- 스탬프 언어: 잉크 스탬프형 재료 뱃지(안전=초록 스탬프, 테스트중=앰버, 반응=빨강 잉크)를 **도감 그리드와 리포트(음식 여권)에서만** 사용. 다른 화면으로 번지지 않는다.
- clinic-sage 시안에서 채택하는 것은 **콘텐츠 패턴만**: 히어로 카드의 안심 문구("이상 반응 없음"), 큰 숫자 카운트 타일. 색은 가져오지 않는다.
- night-watch(다크 모드)는 **보류** — warm 시스템 light-only 유지.
- 탈락/보류 시안 원본: `docs/mocks/proposals/` (로컬, 미커밋).
기준 시안은 `docs/mocks/hybrid/`(1차) + `docs/mocks/warm-kr/`(구판 참고), 정보 구조 참고용 대안은 `docs/mocks/clinical-en/` — 커버리지와 외부 레퍼런스는 `docs/mocks/README.md`.
아래 clinic 토큰은 **레거시**로, 아직 마이그레이션 안 된 화면에서만 유지한다. **새 작업은 warm-\* 토큰만 쓴다.**

변하지 않는 핵심 원리: **알레르기 상태 = 신호등.** 색이 곧 기능이다.
- 안전/테스트 완료 → 초록 · 테스트 중/대기 → 노랑 · 반응 확정 → 빨강
- 신호등 시맨틱 토큰(safe/testing/reaction)은 warm-kr에서도 그대로 쓴다. 색 단독 의미 전달 금지(아이콘+텍스트 병행)도 불변.

- 그림자: `--shadow-warm`(테라코타 틴트 소프트 섀도, `shadow-warm` 유틸). 카드가 보더 대신 이 그림자에 기댄다.
- warm 시스템은 **light-only**(`.dark` 변형 없음).
- 브랜드(테라코타) vs CTA(코랄) 역할 분리를 지킬 것 — 전부 코랄로 칠하지 않는다.

## 타이포그래피
- 기본 UI 폰트: **Pretendard** 권장(한글 친화·모던·무료). 프로젝트에 이미 있는 Paperlogy/SCDream도 가능.
- 스케일: h1 22 / h2 18 / h3 16 / body 16 / sub 14 / caption 12 (px)
- 굵기: 400(본문) / 500(라벨·제목) / 600(강조 최소한). 과한 볼드 지양.
- 항상 문장형(sentence case). 전부 대문자/Title Case 지양.

## 핵심 컴포넌트
- **StatusChip** — safe/testing/reaction 3종. 둥근 라운드 라벨, 시맨틱 색.
- **Card** — 흰 배경, `#E2E9E5` 0.5px 보더, radius 14, 패딩 13~16.
- **Button(primary)** — `#2D7FF9` 배경 + 흰 텍스트, radius 9~10.
- **MetricTile** — 신호등 카운트(안전/테스트중/반응) 한 줄.
- **ProgressBar** — 관찰 진행(예: 3일 중 2일), 시맨틱 색.
- **BottomNav** — 하단 고정 5탭(`Layout.tsx`, `grid-cols-5`): 메뉴 / 일정 / 홈 / 알레르기 / 커뮤니티. 홈이 중앙, 알레르기는 전용 탭으로 상시 노출(알레르기 강조는 로그인+활성 아기일 때 홈을 대시보드 히어로로 전환해 구현 — `HomeRoute.tsx`).

## 원칙
- **모바일 우선.** iOS 시뮬레이터에서 좋아 보이게.
- **차분하고 명확하게.** 지친 부모 대상 → 정보 밀도 낮추고 여백 충분히.
- **상태를 디자인한다.** happy path뿐 아니라 빈 상태/로딩/에러를 반드시.
- **접근성.** 색만으로 의미 전달 금지(아이콘/텍스트 병행), 충분한 대비, 11px 미만 금지.
- **일관성.** 토큰 먼저 잡고 컴포넌트로, 그다음 화면. 일회성 하드코딩 색 금지.
